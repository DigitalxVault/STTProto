# Phase 4.1 Plan 01: Bottom-Anchored Transcript Layout Summary

**One-liner:** Fixed bottom-bar with column-reverse flex transcript panel anchors entries to bottom-left, growing upward, beside mic button at bottom-right.

## What Was Built

Restructured the HTML and CSS to move the transcript panel and mic button into a shared `bottom-bar` div fixed to the viewport bottom. The transcript uses `flex-direction: column-reverse` so new entries naturally stack upward. The mic button remains right-aligned. Safe-area padding consolidated to the bottom-bar. Dead `.state-demo` CSS removed.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Container strategy | Single `.bottom-bar` fixed wrapper | Unified safe-area handling, spatial sync |
| Transcript ordering | `column-reverse` flex | Zero-JS upward growth with native scroll |
| Event passthrough | `pointer-events: none` on bar, `auto` on children | No tap-blocking over page content |
| Demo strip | Removed `.state-demo` CSS | Dead code from Phase 1 verification |
| Transcript appearance | Transparent bg, text-shadow | Clean aesthetic with legibility on dark bg |
