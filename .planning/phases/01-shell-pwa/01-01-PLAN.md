Create index.html at project root with:

HEAD section (from research Pattern 2):

- viewport: width=device-width, initial-scale=1.0, viewport-fit=cover

- Inside .mic-container: a single mic button (.mic-btn) with an SVG microphone icon inside
- For Phase 1 verification: add a small demo area (can be hidden or at top) showing all three button states side-by-side so the checker can verify idle/recording/processing render correctly. Use a comment noting this is for Phase 1 verification only.
- Script tag loading app.js at end of body (defer or before closing body tag)

The mic button should default to class "state-idle". The demo area should show one button with "state-idle", one with "state-recording", one with "state-processing".

SVG mic icon: a simple microphone silhouette (path-based, inline SVG, ~24x24 viewBox, white/light grey fill).

<task type="auto">
  <name>Task 2: Create style.css with military aesthetic and all mic button states</name>
  <files>style.css</files>
  <action>
Create style.css at project root implementing the full military aesthetic and layout.

CSS Custom Properties (from research Pattern 7):
- --bg-primary: #0a0a0a
- --bg-panel: #111111
- --border-dim: #2a2a2a
- --text-primary: #c8c8c8
- --text-dim: #555555
- --accent-red: #8b0000
- --accent-green: #1a4a1a

Reset: box-sizing border-box on *, margin/padding 0.

html, body: height 100%, width 100%, overflow hidden.

body: min-height 100dvh (NOT 100vh -- research warns about iOS Safari toolbar overlap), background var(--bg-primary), color var(--text-primary), font-family "Courier New", Courier, "Lucida Console", monospace, font-size 14px, line-height 1.5, letter-spacing 0.05em, display flex, flex-direction column, padding-bottom env(safe-area-inset-bottom, 16px), padding-top env(safe-area-inset-top, 0px).

Header/status bar: uppercase, letter-spacing 0.15em, font-size 11px, color var(--accent-green) or #3a7a3a for phosphor-screen feel, padding ~1rem, border-bottom 1px solid var(--border-dim).

.transcript-panel: flex 1, overflow-y auto, padding 1rem, background var(--bg-panel), border 1px solid var(--border-dim), margin ~0.5rem, font-size 13px.

.mic-container: flex-shrink 0, display flex, justify-content flex-end (RIGHT-aligned per user reference), align-items center, padding 1.5rem. The button should sit in the bottom-right corner.

Mic button base (.mic-btn): width 80px, height 80px, border-radius 50%, border 3px solid #555, background #2a2a2a, cursor pointer, display flex, align-items center, justify-content center, position relative, transition background 0.2s and border-color 0.2s. Add a subtle metallic/silver gradient or box-shadow to match the reference image's metallic circular appearance (e.g., background: radial-gradient(circle at 30% 30%, #4a4a4a, #2a2a2a)).

Three states (from research Pattern 6):
1. .mic-btn.state-idle: background metallic gradient, border-color #555
2. .mic-btn.state-recording: background #8b0000, border-color #cc0000, animation pulse-red 1.2s ease-in-out infinite
3. .mic-btn.state-processing: background #1a1a1a, border-color #444, with ::after spinner pseudo-element

@keyframes pulse-red: 0% box-shadow 0 0 0 0 rgba(200,0,0,0.6), 70% box-shadow 0 0 0 16px rgba(200,0,0,0), 100% box-shadow 0 0 0 0 rgba(200,0,0,0).

@keyframes spin: to transform rotate(360deg). The ::after on .state-processing: content "", position absolute, width 40px, height 40px, border 3px solid #333, border-top-color #888, border-radius 50%, animation spin 0.8s linear infinite.

Mic icon inside button: SVG fill should be #c8c8c8 (light) for idle, white for recording, hidden or dimmed for processing.

Demo state area styling: small horizontal row, can be subtle, used for verification.
