     "orientation": "portrait",
     "theme_color": "#1a1a1a",

     "prefer_related_applications": false,
     "icons": [
       {
         "src": "/icons/icon-192.png",
         "type": "image/png",

       }
     ]
   }

<task type="auto">
  <name>Task 2: Create sw.js service worker with precache and cache-first strategy</name>
  <files>sw.js</files>
  <action>
Create sw.js at project root implementing the cache-first app shell pattern (from research Pattern 3).

The service worker must:

1. Define CACHE_NAME as "rt-trainer-v1" (version in name enables cache busting on updates).

2. Define PRECACHE_URLS array listing all static assets:
   - "/"
   - "/index.html"
   - "/style.css"
   - "/app.js"
   - "/manifest.json"
   - "/icons/icon-192.png"
   - "/icons/icon-512.png"
   - "/icons/apple-touch-icon.png"

3. Install event handler:
   - Open cache by CACHE_NAME
   - cache.addAll(PRECACHE_URLS) to precache everything
   - self.skipWaiting() to activate immediately without waiting for old tabs to close

4. Activate event handler:
   - Delete any old caches (keys that don't match current CACHE_NAME)
   - self.clients.claim() to take control of all open pages immediately

5. Fetch event handler:
   - Only intercept GET requests
   - Only intercept same-origin requests (check url.origin === location.origin)
   - Do NOT intercept /api/* requests (future Phase 3 proxy must bypass cache)
   - Cache-first strategy: check cache, if miss then fetch from network, cache the response, return it
   - If both cache and network fail, return Response.error()

Add the /api/* exclusion now to avoid a hard-to-debug issue in Phase 4 where the service worker caches API responses. Use: `if (url.pathname.startsWith("/api/")) return;`

Follow the exact code structure from research Pattern 3 but add the /api/ exclusion.
  </action>
  <verify>
- `cat sw.js` shows install/activate/fetch handlers
- Grep for "PRECACHE_URLS" confirms asset list
- Grep for "/api/" confirms API route exclusion
- Grep for "skipWaiting" and "clients.claim" confirms immediate activation
  </verify>
  <done>sw.js exists with precache on install, old cache cleanup on activate, cache-first fetch for same-origin GET requests excluding /api/*, and immediate activation via skipWaiting + clients.claim.</done>
</task>
