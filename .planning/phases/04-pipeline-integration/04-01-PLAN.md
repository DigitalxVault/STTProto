<task type="auto">
  <name>Task 1: Wire audio-captured event to /api/transcribe and transcript panel</name>
  <files>app.js</files>
  <action>
Make these changes to app.js in order:

1. ADD a transcriptPanel DOM reference in the "DOM References" section (after line 14):
   ```
   const transcriptPanel = document.querySelector('.transcript-panel');
   ```

2. DELETE the `setState('idle')` call on line 131 inside `recorder.onstop`. Keep the empty-blob guard's own `setState('idle')` on line 122 — that path never dispatches the event. Add a comment: `// Phase 4: idle transition owned by audio-captured listener`

3. ADD these three functions after the Diagnostics section (bottom of file), in a new section headed `// -- Pipeline Integration`:

   a) `addTranscriptEntry(text)`:
      - Query `.transcript-placeholder` inside transcriptPanel; if found, remove it
      - Create a `<p>` element, set `textContent` to the text argument (NEVER innerHTML — XSS safety)
      - Add class `transcript-entry` to the element
      - Append to transcriptPanel
      - Auto-scroll: `transcriptPanel.scrollTop = transcriptPanel.scrollHeight`

   b) `addTranscriptError(label)`:
      - Same as addTranscriptEntry but creates `<p>` with class `transcript-error`
      - textContent set to `[ERR] ${label}`
      - Also calls `showStatusTemp(label, '#cc4444', 4000)` for the status bar

   c) `async transcribeAndDisplay(blob)`:
      - Build FormData: `formData.append('file', blob, 'audio.' + (blob.type.includes('mp4') ? 'mp4' : 'webm'))`
      - Do NOT set any headers — let browser set Content-Type with boundary
      - Wrap `fetch('/api/transcribe', { method: 'POST', body: formData })` in try/catch
      - catch block: `console.error('[STT] Network error:', err)`, call `addTranscriptError('NETWORK ERROR')`, call `setState('idle')`, return
      - After fetch, check `response.ok`. If false: parse body with `.json().catch(() => ({}))`, log `[STT] API error: ${response.status}`, call `addTranscriptError('TRANSCRIPTION FAILED')`, call `setState('idle')`, return
      - On success: `const { text } = await response.json()`. If text is truthy, call `addTranscriptEntry(text)`. Call `setState('idle')`.

4. ADD event listener at the bottom of the file:
   ```
   document.addEventListener('audio-captured', function (e) {
     transcribeAndDisplay(e.detail.blob);
   });
   ```

5. Update the diagnostics log to include: `console.log('[STT] Pipeline integration loaded');`

IMPORTANT things to avoid:
- Do NOT set Content-Type header on the fetch call (breaks multipart boundary)
- Do NOT use innerHTML or insertAdjacentHTML for transcript entries (XSS risk from API text)
- Do NOT add setState('processing') inside transcribeAndDisplay — stopRecording() already sets it before onstop fires
  </action>
