<task type="auto">
  <name>Task 2: Implement PTT gesture + MediaRecorder lifecycle in app.js</name>
  <files>app.js</files>
  <action>
    Rewrite app.js to add PTT audio capture while preserving the existing service worker registration at the top. The file should be structured as a single IIFE or top-level scope with these concerns:

    **1. DOM references:**
    - `const micBtn = document.querySelector('.mic-btn');`

    **2. State variables (module-scoped):**
    - `let stream = null;` — cached MediaStream (acquired once, reused)
    - `let recorder = null;` — current MediaRecorder instance
    - `let chunks = [];` — audio data chunks

    **3. Helper: setState(state)**
    - Accepts 'idle', 'recording', or 'processing'
    - Removes all three state classes (`state-idle`, `state-recording`, `state-processing`) from micBtn
    - Adds `state-${state}` class
    - Updates `.status-indicator` text: idle = "STANDBY", recording = "REC", processing = "PROCESSING"

    **4. Helper: acquireMic()**
    - If `stream` is already set and tracks are live, return it immediately
    - Otherwise call `navigator.mediaDevices.getUserMedia({ audio: { channelCount: 1, echoCancellation: true, noiseSuppression: true, autoGainControl: true } })`
    - Cache result in `stream`
    - Return stream
    - On error: log to console, return null

    **5. Helper: startRecording(mediaStream)**
    - Determine mimeType using a priority list checked with `MediaRecorder.isTypeSupported()`:
      1. 'audio/webm;codecs=opus'
      2. 'audio/webm'
      3. 'audio/mp4'
      4. '' (empty string = browser default)
    - Create `new MediaRecorder(mediaStream, { mimeType })` (omit mimeType key entirely if empty string)
    - Set `chunks = []`
    - `recorder.ondataavailable = (e) => { if (e.data?.size > 0) chunks.push(e.data); }`
    - `recorder.onstop = () => { ... }` — see step 6
    - `recorder.start()` — no timeslice argument
    - Call `setState('recording')`

    **6. recorder.onstop handler:**
    - Build blob: `const blob = new Blob(chunks, { type: recorder.mimeType });`
    - Log to console: `console.log('Audio captured:', { size: blob.size, type: blob.type })`
    - Guard: if `blob.size === 0`, log warning and call `setState('idle')`, return
    - Dispatch `new CustomEvent('audio-captured', { detail: { blob } })` on `document`
    - Call `setState('idle')` (Phase 4 will change this to 'processing' before sending to API)

    **7. Helper: stopRecording()**
    - Guard: `if (!recorder || recorder.state !== 'recording') return;`
    - `recorder.stop()`

    **8. Pointer Event listeners on micBtn:**
    - `pointerdown`: call `e.preventDefault()`, call `micBtn.setPointerCapture(e.pointerId)`, call `handlePressStart()`
    - `pointerup`: call `handlePressEnd()`
    - `pointercancel`: call `handlePressEnd()` — treat identically to pointerup

    **9. handlePressStart():**
    - `const mediaStream = await acquireMic();`
    - If null (permission denied), return without state change
    - Call `startRecording(mediaStream)`

    **10. handlePressEnd():**
    - Call `stopRecording()`
    - Note: setState('idle') happens in onstop handler, not here

    **Important implementation notes:**
    - pointerdown handler must be async (or call an async function) because acquireMic is async
    - Keep the existing `if ('serviceWorker' in navigator)` block at the top of the file
    - Use `'use strict';` at the top
    - All new code should be vanilla JS, no modules, no build step
    - Do NOT use `addEventListener('touchstart', ...)` or `addEventListener('mousedown', ...)` — Pointer Events only
