<task type="auto">
  <name>Task 1: Add error handling, permission UX, and blob validation</name>
  <files>app.js</files>
  <action>
    Enhance app.js from Plan 02-01 with these specific additions:

    **1. Permission denial feedback:**
    - In `acquireMic()`, when getUserMedia fails:
      - If error.name === 'NotAllowedError' or 'PermissionDeniedError': update `.status-indicator` to show "MIC BLOCKED" in red for 3 seconds, then revert to "STANDBY"
      - For other errors (NotFoundError, etc.): update `.status-indicator` to show "MIC ERROR" for 3 seconds
      - Log the full error to console in all cases
      - Set `stream = null` so next press retries acquisition

    **2. MediaRecorder error handling:**
    - Add `recorder.onerror = (e) => { console.error('MediaRecorder error:', e.error); setState('idle'); }` in startRecording()
    - Wrap `recorder.start()` in try/catch — if start fails, log error and setState('idle')

    **3. Blob validation enhancement in onstop:**
    - After building blob, log detailed info: `console.log('[PTT] Blob ready:', { size: blob.size, type: blob.type, chunks: chunks.length })`
    - If blob.size === 0: log `console.warn('[PTT] Empty blob — discarded')` and setState('idle'), do NOT dispatch event
    - If blob.size > 0: dispatch CustomEvent and log `console.log('[PTT] Dispatched audio-captured event')`

    **4. Stream health check:**
    - In acquireMic(), when stream is cached, verify tracks are still live: `stream.getAudioTracks().every(t => t.readyState === 'live')`
    - If any track is ended, set `stream = null` and re-acquire (iOS can end tracks when app backgrounds)

    **5. MIME type logging:**
    - In startRecording(), after determining mimeType, log: `console.log('[PTT] Using MIME:', mimeType || 'browser default')`
    - This confirms which codec path was selected on each browser

    **6. Rapid tap guard:**
    - In handlePressEnd(), add check: if recorder is null or state is not 'recording', just call setState('idle') and return
    - This prevents errors when user taps faster than mic acquisition completes

    **Important:**
    - Do NOT restructure or rewrite the entire file. Make surgical additions to the existing functions from Plan 02-01.
    - Preserve all existing functionality — this is additive hardening only.
    - All status indicator text changes should use the same status-indicator element targeted in setState()
  </action>
  <verify>
    1. Deny mic permission when prompted — status indicator should flash "MIC BLOCKED" for 3 seconds, then revert to "STANDBY"
    2. Grant permission, press and hold, release — console shows `[PTT] Blob ready: { size: NNN, type: '...', chunks: N }` and `[PTT] Dispatched audio-captured event`
    3. Console shows `[PTT] Using MIME: audio/webm;codecs=opus` (or equivalent for current browser)
    4. Rapid tap does not throw errors or leave button stuck in recording state
    5. Listen for CustomEvent: run `document.addEventListener('audio-captured', e => console.log('GOT IT', e.detail.blob.size))` in console before pressing — should fire with size > 0
  </verify>
  <done>
    Permission denial shows visible feedback. All blobs are validated before dispatch.
    MIME type selection is logged for debugging. Rapid taps are gracefully handled.
