Key facts from research:
- Web Standard export: `export async function POST(request)` — NOT legacy req/res
- Native `request.formData()` for multipart parsing — no library needed
- `toFile()` from `openai` SDK wraps Buffer for Whisper — no filesystem needed
- Filename extension in toFile() is CRITICAL — Whisper uses it for format detection
- MIME-to-extension map: audio/webm -> .webm, audio/mp4 -> .mp4, audio/ogg -> .ogg, audio/wav -> .wav, audio/mpeg -> .mp3
- Vercel body limit 4.5 MB — enforce 4 MB in the function
- package.json needs `"type": "module"` for import syntax
- vercel.json needs `functions.api/transcribe.js.maxDuration: 30`
- CORS not needed (same-origin deployment)
</context>

<task type="auto">
  <name>Task 2: Create api/transcribe.js and update vercel.json</name>
  <files>api/transcribe.js, vercel.json</files>
  <action>
    Create api/transcribe.js as a Web Standard serverless function with named POST export:

    ```javascript
    import OpenAI, { toFile } from 'openai';

    export async function POST(request) { ... }
    ```

    The function must implement these guards IN ORDER:
    1. Check `process.env.OPENAI_API_KEY` exists — return 500 `{ error: "Server configuration error" }` if missing
    2. Check content-type starts with `multipart/form-data` — return 400 `{ error: "Expected multipart/form-data" }` if wrong
    3. Parse `request.formData()` in try/catch — return 400 `{ error: "Failed to parse form data" }` on failure
    4. Get `file` from formData — return 400 `{ error: "No audio file provided" }` if missing or is a string
    5. Check `file.size > 4 * 1024 * 1024` — return 413 `{ error: "Audio file exceeds 4 MB limit" }` if too large

    Then the core logic:
    1. Build MIME-to-extension map: audio/webm->webm, audio/mp4->mp4, audio/ogg->ogg, audio/wav->wav, audio/mpeg->mp3. Default to "webm" if unknown.
    2. Instantiate `new OpenAI()` (reads OPENAI_API_KEY from env automatically)
    3. Convert file to ArrayBuffer, then Buffer.from(arrayBuffer)
    4. Call `openai.audio.transcriptions.create({ file: await toFile(buffer, 'audio.EXT', { type: file.type }), model: 'whisper-1' })`
       - The filename `audio.EXT` with correct extension is CRITICAL — Whisper uses it for format detection
    5. Return `Response.json({ text: transcription.text })`
    6. Wrap the Whisper call in try/catch — on error, return `Response.json({ error: err.message ?? 'Transcription failed' }, { status: err.status ?? 500 })`

    DO NOT:
    - Use legacy req/res pattern (that is Next.js pages router only)
    - Use fs.createReadStream (no filesystem in serverless)
    - Add CORS headers (same-origin deployment)
    - Hard-code any API key

    Update vercel.json to ADD the `functions` key alongside the existing `headers` key:
    ```json
    {
      "headers": [ ... existing sw.js header ... ],
      "functions": {
        "api/transcribe.js": {
          "maxDuration": 30
        }
      }
    }
    ```
    Preserve the existing headers array exactly as-is.
  </action>
